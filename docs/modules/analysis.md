# analysis

Parallel project status analysis (`dcp/cortex/analysis.py` + `dcp/agents/status_agent.py`).

## Flow

1. `AnalysisManager.start(paths?)` — omitted paths = every registered project. One batch at a time (409 semantics like [[sentry]]'s ScanManager).
2. `ThreadPoolExecutor` (`DCP_ANALYSIS_WORKERS`, default 4) fans out; per project:
   - stage inference + context assembly (reuses [[cortex]])
   - `StatusReportAgent.build_status()` — Tier 1 LangGraph over the project digest (README head, manifest, file tree, git log; size-capped) when AI is configured, deterministic Tier 0 otherwise. Internal fallback: never raises for missing AI.
   - writes `<project>/project_status.md` (starts with `<!-- generated by Developer Control Plane -->`)
   - updates `projects.status_headline/status_health/analyzed_at`, logs a `ProjectStatusReport` inference + `StatusFileWritten` decision
3. Live progress: `{running, total, done, failed, current[], errors{}}`. One project failing never kills the batch.

## Endpoints

| Method | Path | Notes |
| --- | --- | --- |
| POST | `/api/analysis/start` | `{paths?: []}`; 409 if running |
| GET | `/api/analysis/status` | batch progress |
| POST | `/api/analysis/project?path=` | analyze one, synchronous |
| GET | `/api/analysis/report?path=` | raw project_status.md |

UI: Status column (health dot + headline) on the Projects tab, "Analyze All" with progress line, per-project Analyze in the detail panel.
